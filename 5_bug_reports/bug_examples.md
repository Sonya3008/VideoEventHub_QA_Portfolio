**BUG-01. Видеопоток не восстанавливается после кратковременного обрыва сети**

Шаги воспроизведения:

1. Запустить live-видео с камеры
2. Отключить сеть на 10 секунд
3. Включить сеть

Фактический результат:
Live-видео не восстанавливается, требуется ручной перезапуск сервиса.

Ожидаемый результат:
Система должна автоматически восстановить видеопоток.

Severity: Critical
Priority: High

Комментарий:
Приводит к полной потере наблюдения до вмешательства оператора.

Вложения: 
- логи
- скринкасты


**BUG-02. Отсутствует логирование обрыва видеопотока**

Шаги воспроизведения:

1. Запустить live-видео с камеры
2. Отключить сеть на 10 секунд
3. Включить сеть

Фактический результат:
В логах отсутствует запись об обрыве соединения.

Ожидаемый результат:
Обрыв видеопотока должен логироваться с указанием камеры и времени.

Severity: Major
Priority: Medium

**BUG-03. Событие не сохраняется при активном видеопотоке**

Фактический результат:
Видео доступно, событие отсутствует в UI и БД.

Ожидаемый результат:
Событие должно сохраняться и отображаться.

Severity: Critical
Priority: High

Комментарий:
Система визуально выглядит рабочей, но фактически теряет данные.

**BUG-04. API возвращает 200 OK при некорректном теле события**

Шаги:

1. Отправить событие без обязательного параметра timestamp

Фактический результат:
API возвращает 200 OK, событие не сохраняется.

Ожидаемый результат:
API должен вернуть 400 Bad Request с описанием ошибки.

Severity: Major
Priority: Medium
